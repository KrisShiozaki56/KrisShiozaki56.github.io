User Flows

I created a high-fidelity user flow diagram using the final mockups, which captures the two major task flows outlined earlier. The purpose of this diagram is to guide the developers to ensure the UX is implemented as planned. It also helps everyone to understand the user's journey in our app & allows us to identify edge cases.

The next steps for the user flow diagram will be to add brief notes regarding animations & transitions underneath the mockups, as well as include screens, which tackle edge cases.

User Testing Plan

I worked with a UX design researcher at Connected, where she helped me create a user testing plan. Unfortunately, the plan itself cannot be shared due to branding regulations, but the plan outline included the following:

  • Objective of Test Session
  • Summary of Interview Plan (list)
  • Introduction (briefly introduce ourselves & the project)
  • Exploring Problem Space (gain an understanding of the users' related experience to the product)
  • Usability Test Cases Outline (Brief description & goals for each test)
  • Setting the Stage (Have the user imagine different scenarios)
  • 2 Test cases (Goal, Task, Observations, Discussion questions)
  • Conclusion (thank you & final questions)

Unfortunately, my co-op term ended before I was able to have some user testing sessions with my co-workers but now if any members of our team would like to conduct a user testing session, the documentation is there!

Reflection

This project was many of my firsts - my first time taking on a design role in a long-term team project, my first time creating user flows, my first co-op term, etc. It was a wonderful learning experience and I was able to take away invaluable skills. The next steps for this project include completing the development of the application, implementing more creative features & metrics into the application & conducting user testing. I hope to one day hold a user testing session!
